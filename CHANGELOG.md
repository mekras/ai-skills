# История изменений

Все заметные изменения в этой коллекции навыков отражаются в этом файле.

Формат основан на `Keep a Changelog`, а версии следуют `SemVer`.

## [Не выпущено]

## [0.5.0] - 2026-05-27

### Добавлено

- Добавлено семейство навыков `requirements-*` для выявления, анализа,
  спецификации, проверки и управления требованиями к ПО.

### Изменено

- Уточнены границы `requirements-analysis`: анализ черновых сведений отделён
  от проверки готового набора требований.

## [0.4.0] - 2026-05-27

### Добавлено

- Добавлен навык `project-documentation` для формулирования и проверки
  концепции проекта как исходной проектной документации.

## [0.3.0] - 2026-05-27

### Добавлено

- Добавлен навык `software-architecture` для проектирования архитектуры ПО,
  подготовки ADR, проверки компонентных границ и управления сложностью системы.
- Добавлена зависимость `talkstream/ru-text` для базового редакторского слоя
  навыков `ru-lang` и `ru-dev`.

### Изменено

- Для `ru-lang` и `ru-dev` включено неявное срабатывание в метаданных OpenAI.
- Расширены отрицательные проверки срабатывания `software-architecture` для
  близких задач без архитектурной неопределённости.

### Исправлено

- Исправлена формулировка обязательного раздела в `project-readme`: «установке
  или подключению».

## [0.2.2] - 2026-05-22

### Изменено

- Усилен навык `ru-lang`: область применения теперь явно включает собственные
  промежуточные и итоговые ответы агента.
- Добавлена финальная проверка русскоязычных отчётов и комментариев на
  англо-русские кальки.
- Правило для гибридных технических терминов обобщено: навык должен ловить
  конструкции вида `{латиница}-{русское слово}` и `{бренд}-{русское слово}`,
  а не только заранее перечисленные примеры.

### Добавлено

- Добавлен проверочный случай `ru-lang` для гибридных технических терминов.
- В базе знаний зафиксирована ошибка от 2026-05-22: агент может иметь доступ к
  языковому навыку, но не применить его к собственному итоговому ответу.

## [0.2.1] - 2026-05-19

### Исправлено

- Уточнён навык `project-rules-audit`: запрещено использовать в доказательствах
  устаревшие строки из памяти и прежних версий файлов.
- В `references/audit-checklist.md` добавлен раздел «Доказательства» и поле
  `Доказательство` в формат вывода, чтобы `evidence` опирался только на текущее
  содержимое файлов.
- В `knowledge/skill-development.md` добавлено наблюдение о типовой ошибке с
  устаревшими строками в `evidence` и требование перепроверять текущий снимок
  файла перед итогом.

## [0.2.0] - 2026-05-19

### Добавлено

- Добавлен корневой манифест `APM` в `apm.yml`.
- Добавлен `README.md` с инструкциями по установке через `APM`.
- Добавлен `LICENSE` с лицензией `MIT`.
- Добавлен пилотный сценарий проверки `APM` в `docs/apm-pilot.md`.
- Добавлен навык `project-readme` для сопровождения русскоязычного
  `README.md` проекта.
- Добавлен навык `primary-data-pipeline` для оркестрации конвейера первичных
  данных.
- Добавлен навык `primary-data-summary` для сводки состояния первичных данных.
- Добавлен навык `source-fact-extraction` для извлечения проверяемых фактов из
  первичных данных.

### Изменено

- Репозиторий переведён в формат коллекции навыков `APM`.
- Навыки перенесены в структуру `skills/<name>/`.
- Обновлены внутренние правила и база знаний под новую структуру.
- Расширены `source-inventory` и `source-impact-audit`: добавлены переносимые
  контракты слоя первичных данных, проверки индексов и правила работы с
  извлечёнными фактами.

## [0.1.0] - 2026-05-15

### Добавлено

- Начальная публикуемая версия коллекции навыков в формате `APM`.
