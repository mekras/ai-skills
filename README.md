# Коллекция навыков для агентов ИИ

Это репозиторий переносимых навыков для агентной разработки и сопровождения
проектов. Коллекция предназначена для установки через `APM` и помогает
подключать к `Codex`, `Claude Code` и совместимым средам единый набор навыков
для русскоязычных текстов, проектной документации, аудита правил, работы с
источниками и других повторяемых задач.

## Общее описание

В репозитории собраны публикуемые навыки в формате коллекции `APM`.
Каждый навык хранится отдельно и содержит:

- `SKILL.md` с триггерами, порядком работы и ограничениями;
- `agents/openai.yaml` с метаданными интерфейса;
- `evals/triggers.json` с проверками срабатывания;
- при необходимости `references/`, `scripts/`, `assets/`.

Текущая коллекция включает:

- `private-project-knowledge` — локальные знания проекта вне VCS;
- `project-readme` — сопровождение русскоязычного `README.md` проекта;
- `project-rules-audit` — аудит правил проекта для агента;
- `ru-dev` — русский текст в разработке ПО;
- `ru-lang` — базовые правила русского текста;
- `skill-development` — создание и сопровождение навыков;
- `source-impact-audit` — анализ влияния источников на документацию;
- `source-inventory` — учёт и техническая синхронизация внешних источников;
- `transcript-analysis` — разбор и чистка стенограмм.

Репозиторий распространяется по лицензии `MIT`.

## Установка и подключение

Основной способ использования — установка коллекции через `APM` в
пользовательское окружение.

Установить всю коллекцию:

```bash
apm install -g github.com/mekras/ai-skills#master
```

Установить только выбранные навыки:

```bash
apm install -g github.com/mekras/ai-skills#master --skill ru-lang --skill ru-dev
```

Обновить уже установленную коллекцию:

```bash
apm update -g
```

Проверить, есть ли новые версии зависимостей:

```bash
apm outdated -g
```

Коллекция ориентирована прежде всего на `Codex` и `Claude Code`. Установка
идёт напрямую из Git через `apm install`; команда `apm pack` для этого
репозитория пока не используется.

## Использование

После установки навыки становятся доступны агенту как обычные локальные
навыки. Дальше их можно вызывать явно по имени или использовать неявное
срабатывание там, где оно поддерживается средой.

Типовые сценарии:

- использовать `ru-lang` и `ru-dev` для русскоязычных ответов, сообщений
  коммитов и технических отчётов;
- использовать `project-readme` для создания и проверки `README.md` проекта;
- использовать `project-rules-audit` для проверки `AGENTS.md`, `CLAUDE.md` и
  других правил агента;
- использовать `skill-development` для создания новых навыков и проверки
  существующих;
- использовать `source-inventory` и `source-impact-audit` для проектной работы
  с внешними источниками.

Минимальный сценарий проверки на машине разработчика:

1. Установить `APM`.
2. Выполнить `apm install -g github.com/mekras/ai-skills#master`.
3. Убедиться, что нужные навыки видны в используемом клиенте.
4. Выполнить `apm update -g`, когда нужно подтянуть свежие изменения из ветки
   `master`.

Если нужна только часть набора, установите её через повторяющийся флаг
`--skill`.

## Разработка

Публикуемые навыки лежат в `skills/<name>/`. Общая методика и знания
репозитория находятся вне коллекции навыков:

- `AGENTS.md` — правила работы агента в этом репозитории;
- `knowledge/` — устойчивая база знаний по проектированию навыков;
- `tools/` — служебные инструменты проверки.

При добавлении или изменении навыка:

1. Обновите `SKILL.md`.
2. Синхронизируйте `agents/openai.yaml`, если меняется назначение навыка.
3. Добавьте или обновите `evals/triggers.json`.
4. При заметных изменениях обновите [CHANGELOG.md](./CHANGELOG.md).

Локальная проверка структуры триггеров:

```bash
python3 tools/validate-trigger-evals.py --require-all
```

Та же проверка через `APM`:

```bash
apm run validate-triggers
```

Коллекция использует `SemVer`, но пока команда обновляется по текущему
состоянию ветки `master`. Для воспроизводимых установок позже стоит перейти
на теги или фиксированные версии.
