# Пилот APM

Этот сценарий нужен, чтобы быстро проверить пользовательский путь установки
и обновления коллекции навыков.

## Цель

Проверить, что разработчик может:

- установить коллекцию навыков в пользовательское окружение;
- обновить её одной командой;
- увидеть ключевые навыки в `Claude Code` и `Codex`.

## Шаги

1. Установить `APM`.
2. Выполнить:

```bash
apm install -g github.com/mekras/ai-skills#master
```

3. Проверить установленные зависимости:

```bash
apm deps list -g
```

4. Обновить коллекцию:

```bash
apm deps update -g
```

Если нужно обновить сам CLI `APM`, используйте `apm self-update`.

5. Проверить, что агенту доступны навыки:

- `ru-lang`
- `ru-dev`
- `skill-development`
- `project-rules-audit`

## Что считать успехом

- `APM` завершает установку без ошибок.
- Коллекция видна в списке зависимостей пользователя.
- После обновления `APM` не требует ручной синхронизации по проектам.
- Навыки доступны в поддерживаемых клиентах.

## Известные ограничения

- Ветка `master` даёт быстрые обновления, но ухудшает воспроизводимость.
- Реальный путь размещения навыков зависит от целевого клиента и версии
  `APM`.
