# project-readme

Помогай сопровождать `README.md` проекта так, чтобы он быстро объяснял суть
проекта, способ начала работы, использование и правила разработки.

## Обязательная зависимость

- Используй `ru-lang` как базовый навык для русского текста.
- Если README включает разработческую часть с заметным объёмом технических
  формулировок, дополнительно применяй `ru-dev`.

## Обязательный порядок

1. Сначала пойми, что это за проект, для кого он нужен и что пользователь
   хочет сделать: создать новый `README.md`, переписать существующий, закрыть
   пробелы или проверить структуру.
2. Перед правкой собери факты из репозитория: название проекта, назначение,
   способ установки или подключения, команды запуска, сценарии использования,
   путь разработки. Не выдумывай команды, зависимости, интерфейсы и статусы.
3. Первая строка `README.md` должна быть заголовком первого уровня на русском
   языке.
4. Если достоверно определить название проекта нельзя, не угадывай его и
   спроси пользователя.
5. Пиши README по-русски.
6. Убедись, что файл содержит все обязательные части:
   - краткое общее описание: что это и для чего нужно;
   - руководство по установке или подключению;
   - подробную документацию по использованию;
   - руководство по разработке.
7. Если проект маленький, разделы могут быть короткими, но все четыре
   обязательные части должны присутствовать явно.
8. Если каких-то сведений не хватает, сначала попытайся найти их в проекте.
   Спрашивай пользователя только там, где без ответа высок риск придумать
   ложные факты, особенно для названия проекта, установки и сценариев
   использования.
9. После правки быстро перечитай `README.md` как документ для нового
   участника: в первой минуте чтения должно быть понятно, что это за проект,
   как начать работу и где искать команды для разработки.

## Границы

- Не превращай `README.md` в внутренний регламент проекта для агента.
- Не подменяй `README.md` справкой по навыку агента, `AGENTS.md`,
  `CLAUDE.md`, `CHANGELOG.md` или release notes.
- Не оставляй первую строку на английском языке, если задача про русский
  `README.md`.
- Не выдумывай название проекта, если его нельзя уверенно вывести из
  репозитория и контекста пользователя.
- Не пропускай обязательные разделы только потому, что проект кажется
  очевидным.

## Вывод

В отчёте кратко разделяй: какие разделы созданы или изменены, чем подтверждены
факты в README и какие сведения остались непроверенными.
