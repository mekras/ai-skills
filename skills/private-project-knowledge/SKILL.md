# private-project-knowledge

Помогай заводить и применять приватный слой знаний проекта так, чтобы он
помогал агенту работать точнее, но не попадал в файлы под управлением VCS.

## Обязательный порядок

1. Уточни цель, если она не ясна: создать локальный слой, подключить его,
   перенести туда сведения, проверить утечки или применить уже существующие
   знания.
2. Раздели типы сведений:
   - локальные знания и правила для агента;
   - конфиденциальные, но текстовые проектные сведения;
   - настоящие секреты: токены, пароли, приватные ключи, cookie, персональные
     данные с высоким риском.
3. Не записывай настоящие секреты в текстовые файлы правил. Для них предложи
   менеджер секретов, переменные окружения или защищённое хранилище.
4. Если нужно создать локальный слой, сначала проверь VCS и существующие
   исключения. Для Git предпочитай `.git/info/exclude`, если само наличие
   локального слоя не должно попасть в репозиторий.
5. Проверь механизм загрузки. Не добавляй ссылку на локальный файл в tracked
   `AGENTS.md`, если пользователь требует скрыть сам факт существования
   приватных правил.
6. Если авто‑подключение невозможно без следа в VCS, прямо назови это
   ограничение и предложи локальный способ: ручное чтение, локальная
   конфигурация инструмента или неотслеживаемый файл, который пользователь
   явно передаёт агенту.
7. После изменений проверь, что приватные файлы не отслеживаются VCS и не
   появились в staged‑изменениях.

## Что читать дополнительно

- Для вариантов структуры, подключения и проверки утечек читай
  `references/local-layer.md`.

## Границы

- Не обещай скрытность, если путь, имя файла или ссылка записаны в tracked
  файл.
- Не создавай `.gitignore` с приватным путём, если требование — не раскрывать
  даже наличие локального слоя; для Git используй `.git/info/exclude`.
- Не помещай проектные приватные сведения в переносимый навык.
- Не смешивай приватные правила с обычными правилами проекта, если обычные
  правила можно безопасно хранить в VCS.
- Не выводи содержимое приватных файлов в итоговый ответ без явной причины.

## Вывод

В отчёте разделяй: что создано или изменено, что осталось локальным и
неотслеживаемым, чем проверено отсутствие попадания в VCS, какие ограничения
загрузки остаются.
